The most important result of this project is qualitative before it is quantitative: one codebase successfully trained on both a 2D image benchmark and a 3D voxel benchmark with no architecture rewrite between the two tasks. The same model family, training stack, and CLI workflow were reused across both workloads, with task-specific behavior determined by configuration. That is the core empirical validation of the N-dimensional implementation.

For CIFAR-10, the 48-trial sweep completed without failures. The best sweep trial achieved a validation accuracy of 0.7438 and provided the starting point for full retraining. After retraining the selected configuration with the longer schedule, the preserved best validation checkpoint reached a validation accuracy of 0.7334, a top-5 validation accuracy of 0.9708, and a validation loss of 0.8673. The run used 48{,}809{,}188 parameters and took 5{,}741.13 seconds end to end, with the best checkpoint found at epoch 112 before early stopping triggered at epoch 152. The training curves in \autoref{fig:cifar-training} show that the long retraining remained stable, but it did not surpass the short-budget sweep-best validation accuracy from trial 22.

For ModelNet40, the stabilized sweep is informative precisely because it no longer exposes invalid configuration failures. All 15 trials completed successfully, and the best sweep configuration achieved a validation accuracy of 0.8092. Full retraining of that selected configuration did \emph{not} improve on the sweep-best score; instead, the preserved best validation checkpoint reached 0.7573, with a top-5 validation accuracy of 0.9423 and a validation loss of 0.9115. The final model used 16{,}423{,}126 parameters and completed in 918.08 seconds, with the best checkpoint found at epoch 6 and early stopping at epoch 26. This does \emph{not} imply that the architecture fails on 3D data. Rather, it shows that under this particular queue run, the chosen long retraining schedule did not translate the sweep-best configuration into a stronger final checkpoint.

From a software engineering perspective, the queue results are also meaningful. The CIFAR job completed in 126{,}959.45 seconds and the ModelNet40 job completed in 4{,}840.45 seconds, showing that the orchestration layer can execute long multi-stage workflows automatically. The saved logs also show that both final runs used four automatically selected GPUs rather than a manually fixed device list, which demonstrates basic resource awareness in the training pipeline.

Overall, the results support a restrained but clear interpretation. NDSwin-JAX is not yet supported by the kind of repeated, test-set benchmark campaign that would justify strong state-of-the-art claims. Nevertheless, the preserved latest-run artifact set shows that the implementation is functional, configurable, and reusable across dimensionalities. The project succeeds as a practical work in applied machine learning engineering: it translates a known architecture into a broader implementation setting and validates that translation with credible, traceable experiments, while also revealing where retraining settings still need improvement.
