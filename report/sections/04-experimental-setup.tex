The experiments in this report are restricted to the final publishable artifact set preserved under \path{report/data/sources/}. This artifact set combines the repaired CIFAR-10 retraining run from May 18, 2026, the repaired CIFAR-100 sweep-and-retrain run from May 19--20, 2026, and the stabilized ModelNet40 voxel run from April 29, 2026. The goal is to document what was actually built, debugged, and measured, not to mix the strongest-looking numbers from unrelated runs.

The first workload is CIFAR-10, a 10-class image classification task with \(32 \times 32\) RGB inputs \citep{krizhevsky2009cifar}. The second workload is CIFAR-100, which keeps the same input resolution but increases the label space to 100 classes. Both CIFAR loaders use a deterministic stratified validation split from the training set and preserve the official test split for post-training evaluation. The final CIFAR-10 and CIFAR-100 checkpoints were selected by validation accuracy and then recomputed on the held-out test split.

The third workload is a voxelized ModelNet40 export \citep{wu20153dshapenets}. The repository manifest describes 8{,}858 training samples, 985 train-derived validation samples, and 2{,}468 test samples, with one input channel and a spatial resolution of \(32^3\). This experiment uses \texttt{num\_dims = 3}, which activates the generalized 3D path in patch embedding, window partitioning, shifting, and patch merging. A held-out test recomputation is preserved for the final restored checkpoint, but the report remains explicit that the model was selected by validation accuracy.

\paragraph{Selected retraining configurations.}
\begin{itemize}
  \item \textbf{CIFAR-10:} \texttt{num\_dims = 2}, patch size \(2 \times 2\), window size \(4 \times 4\), embed dimension 96, depths \([2,2,18,2]\), batch size 128, learning rate 0.0004300210, weight decay 0.0104108183, drop rate 0.05, drop path 0.1, warmup 20 epochs, cutmix 0.5, color jitter enabled.
  \item \textbf{CIFAR-100:} \texttt{num\_dims = 2}, patch size \(2 \times 2\), window size \(4 \times 4\), embed dimension 96, depths \([2,2,18,2]\), batch size 128, learning rate 0.0002047168, weight decay 0.0267249185, drop path 0.3, warmup 30 epochs, mixup 0.2, cutmix 0.5, cutout size 16.
  \item \textbf{ModelNet40:} \texttt{num\_dims = 3}, patch size \(2 \times 2 \times 2\), window size \(4 \times 4 \times 4\), embed dimension 72, depths \([2,2,6,2]\), batch size 64, learning rate 0.0012622739, weight decay 0.0070498347, drop path 0.1, warmup 3 epochs.
\end{itemize}

Hyperparameter search is performed through configuration-driven random search. The preserved CIFAR-10 sweep evaluated 48 trials with validation accuracy as the selection metric and a 60-epoch budget per trial. The fixed CIFAR-100 sweep evaluated 24 trials with a 50-epoch budget per trial. The ModelNet40 sweep evaluated 15 trials with a 10-epoch budget per trial after the data validation issue in the original queue attempt had been repaired. After sweep selection, each task is retrained with the selected configuration. The CIFAR-10 retraining run was launched with a 600-epoch budget and stopped at epoch 392, restoring the best validation checkpoint from epoch 352. CIFAR-100 was also launched with a 600-epoch budget and stopped at epoch 350, restoring epoch 300. ModelNet40 was launched with a 200-epoch budget and stopped at epoch 27, restoring epoch 7. The restored best state, not the nominal epoch budget, determines the final reported checkpoint.

The experiments were executed through the repository's CLI, sweep, queue, and tmux-backed workflow mechanisms, which are themselves part of the practical contribution. The committed extraction inputs and the published Hugging Face weight repositories are listed in \autoref{app:artifacts}.
